GourmetClip - システム設計書

1. 基本設計 (Basic Design)

1.1. プロダクト概要

GourmetClipは、SNS（Instagram, TikTok等）で見つけた飲食店の紹介動画・画像のスクリーンショットをアップロードするだけで、AIが自動で店舗情報を抽出し、自分だけの「行きたいお店リスト」を作成・管理できるWebアプリケーションです。

1.2. システムアーキテクチャ

個人開発やPoC（概念実証）フェーズに最適化し、サーバーレスかつ無料で運用可能な構成を採用しています。

フロントエンド: React (SPA)

スタイリング: Tailwind CSS, Lucide React (アイコン)

バックエンド・データベース: Google Apps Script (GAS) + Google スプレッドシート

画像解析 (AI): Google Gemini API (gemini-2.5-flash)

ユーザー認証: LocalStorageを利用した端末依存のUUID（擬似ログイン）

1.3. 機能一覧

ユーザー登録: 初回アクセス時にニックネーム、年代、性別を入力し、ローカルに一意のIDを生成。

リスト表示機能: 登録した店舗を都道府県・エリアごとにグループ化して一覧表示。

検索・フィルタリング: 店名・エリア等でのフリーワード検索、および「訪問済/未訪問」での絞り込み。

スクショ解析登録機能: アップロードされた画像をGemini APIに送信し、JSON形式で店舗情報を抽出して自動登録。

ステータス・メモ更新: 訪問済みチェックボックスの切替、およびフリーテキストのメモ入力（自動保存）。

外部連携: 電話発信（tel:）、Webサイト遷移、Google Maps経路案内へのリンク。

2.2. API連携設計 (GASエンドポイント)

2.2.1. データ取得 (GET)

エンドポイント: [GAS_WEB_APP_URL]?userId={userId}

処理内容: スプレッドシートから全データを読み込み、クエリパラメータの userId に一致する行のみをJSON配列として返却する。

2.2.2. データ新規追加 (POST)

リクエストBody: 抽出された店舗情報のJSONオブジェクト

処理内容: action プロパティが存在しない場合、スプレッドシートの最終行に新規データとして追記する。
